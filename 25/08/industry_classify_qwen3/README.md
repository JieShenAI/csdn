

## 任务

需要把每家企业都分类到多个产业链的上游、中游、下游（可以多选）。

## 处理流程

先设计提示词，构建完成alpaca数据集。
再使用llamafactory框架进行推理。
然后把大模型预测的结果上传到mongo db 数据库中。
最后遍历企业名单，从数据库中查询出对应的数据，完成表格的列数据的拼接。

## 思考过程

当时的第一感觉就是把所有的产业链的描述信息都放入进去，但发觉这样会导致提示词过长不可行。
一直在思考有没有一种更简单的方法，我把这个需求也询问了GPT，G PT也没有给我一个可行的方案。
后来我自己想了一个不错的方案，就是把这家企业在每一条产业链上都尝试做一下分类。由于允许该企业属于多条产业链，所以我这种每个类别都尝试的做法是没问题的。
